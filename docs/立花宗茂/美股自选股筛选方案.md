# 美股自选股筛选方案

## 任务目标

整理一份适合纳入统一股票池的大类美股观察池方案，覆盖：

- 常见宽基 ETF
- 常买行业 ETF
- 当前市场关注度较高的美股龙头
- AI、半导体、云软件、消费互联网、金融、能源、医疗、军工航天等重点方向

本文档对应 2026-06-16 的第一版人工初筛结果，目标不是一次性做成最终交易池，而是先形成一份可维护、可扩展、可接数据的 100 只核心观察池。

## 股票池来源

本轮股票池主要来自以下几类来源：

1. 宽基指数与核心 ETF
2. 标普 100 / 标普 500 中的大市值核心成分股
3. 纳指核心科技龙头
4. 半导体与 AI 产业链核心公司
5. 常见行业 ETF 对应的代表性龙头公司
6. 市场热度较高、讨论度高、成交活跃的成长股

说明：

- 用户口头提到“标普 100、500、300”。其中标普 100 与标普 500 是常用来源。
- “标普 300”不是最常见的主流口径。本轮先用“标普 100 + 标普 500 + 纳指核心股 + 主流 ETF”完成首版池子，更适合作为交易底池。
- 若后续需要更完整的美国股票覆盖范围，可再扩展到 Russell 3000 或 CRSP Total Market 口径。

## 筛选口径

本轮纳入标准以“适合长期观察与后续量化处理”为主，核心口径如下：

- 优先纳入市值大、流动性强、成交活跃、资讯覆盖充分的标的
- 优先纳入主流券商和行情源普遍支持的普通股与 ETF
- 优先纳入行业代表性强、机构持仓广、散户认知高的龙头
- 主题股只保留具备较高辨识度或较强交易活跃度的标的
- 尽量避免纳入过小市值、流动性差、事件驱动过强且不稳定的名字

## 不纳入或单独处理的情况

- 状态不稳定的新上市标的：需等交易、数据、流动性稳定后再纳入底池
- 纯概念、低流动性、小票：不进入核心 100 池
- 非标准普通股结构、数据源兼容性差的标的：延后处理
- SpaceX 这类需要单独确认交易状态、ticker、数据可得性的标的：单列观察，不直接并入首版核心 100 池

## 输出结构建议

建议后续统一输出成表格或 CSV，至少包含以下列：

| 字段 | 含义 |
| --- | --- |
| symbol | 交易代码 |
| name | 公司或 ETF 名称 |
| asset_type | `stock` / `etf` |
| source_bucket | 来源池，如 `sp100_core`、`broad_etf`、`ai_semis` |
| sector | 所属大行业 |
| theme | 主题标签，如 `ai`、`semiconductor`、`cybersecurity` |
| watch_priority | `P0` / `P1` / `P2` |
| notes | 纳入原因或特殊说明 |

## 优先级建议

- `P0`：宽基 ETF、核心科技龙头、半导体龙头，优先进入基础池
- `P1`：行业龙头、热门成长股、重要主题 ETF
- `P2`：主题性更强、波动更高、需要继续验证的数据型标的

## 后续执行建议

1. 先按本文档的 100 只名单接入行情与基础面字段
2. 再增加标签体系：宽基、行业、AI、半导体、软件、消费、金融、能源、医疗、军工航天
3. 对 ETF 和个股分开建池，避免后续回测与筛选口径混杂
4. 对 SpaceX 一类特殊标的单开“特殊观察池”，确认数据可得性后再并入
5. 每周或每两周做一次增删审查，避免核心池长期失真
