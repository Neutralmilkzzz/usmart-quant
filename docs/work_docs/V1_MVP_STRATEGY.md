# V1 MVP 策略定义：裸跑实时扫描

## 核心结论

V1 当前不是量化研究系统，也不是回测系统。

V1 当前策略定位是：

> 在已人工筛好的 P0 / P1 美股观察池内，基于 Finnhub 当前可用实时 quote 和基础字段，做最小筛选、简单排序，并把 Top 10 推送给用户人工判断。

换句话说，V1 是一个 **实时扫描与提醒系统**，不是一个已经被验证过的 alpha 策略。

## 不做什么

V1 不做：

- 不做回测
- 不做 beta 因子
- 不做复杂多因子叠加
- 不做质量 / 估值 / 成长混合打分
- 不把基本面字段包装成 alpha
- 不做分钟线技术指标
- 不做 tick / order book 微观结构
- 不输出买卖建议

原因：

- 当前没有回测数据
- 当前 Finnhub key 无法访问 `stock/candle`
- 当前只有实时 quote 和一批基础 metric 字段
- 在没有验证之前，复杂因子只是工程噪音

## 当前策略是什么

### 输入

只使用：

- P0 / P1 股票池
- Finnhub `quote`
- Finnhub `profile2`
- Finnhub `stock/metric` 中少量上下文字段
- Finnhub `market-status`

### 最小过滤

先过滤掉明显不可用的数据：

1. 只保留 `watch_priority in {P0, P1}`
2. 当前价 `c` 必须存在
3. 日涨跌幅 `dp` 必须存在
4. 若可用，保留市值和平均成交量作为展示字段

V1 不因为估值、质量、beta 这类字段做强过滤。

### 简单排序

V1 默认排序：

1. 按 `day_change_pct` 从高到低排序
2. 若涨幅相同，P0 优先于 P1
3. 若仍相同，市值更高者优先

输出 Top 10。

### 结果含义

Top 10 表示：

> 当前观察池内，按实时涨幅排序最靠前的一批高优先级股票。

它不表示：

- 预计上涨
- 应该买入
- 经过回测验证
- 多因子 alpha 更高

## 展示字段

每只入选股票展示：

- `symbol`
- `name`
- `watch_priority`
- `price`
- `day_change_pct`
- `day_change`
- `market_cap`，若可用
- `avg_volume_10d`，若可用
- `reason`

默认 reason：

```text
P0/P1 观察池内当日涨幅靠前
```

如果是 P0：

```text
核心观察标的，且当前涨幅靠前
```

如果市场关闭：

```text
市场当前关闭，结果基于最近可用 quote
```

## 为什么这样做

当前团队更像 QD，不是 QR。

当前任务应该是先把工程闭环跑通：

1. 股票池读取
2. feed 拉取
3. 数据清洗
4. 最小排序
5. Top 10 输出
6. Telegram / Discord 推送
7. 每小时调度
8. AWS 常驻运行

QR 工作，例如因子研究、回测、参数优化、有效性验证，应该是后续独立工作。

## V1 验收口径

V1 只要做到以下几点就算策略层完成：

1. 能读取 P0 / P1 股票池
2. 能拉取 quote
3. 能过滤无效 quote
4. 能按 `day_change_pct` 排序
5. 能返回 Top 10
6. 能清楚标注这不是交易建议

## 后续策略升级

后续如果要做真正策略，需要先补：

1. 可回测历史数据
2. 明确持有周期
3. 明确信号生成时间
4. 明确交易成本假设
5. 明确评价指标
6. 明确样本内 / 样本外验证

在这些条件满足前，不应把复杂因子放进 V1。
